# Code of Conduct

## Posture

This project welcomes contributors who can disagree
substantively without making it personal.

## Expected behaviour

- Disagree about the work, not about the contributor.
- Cite specific text when raising a concern. The framework's
  citation grammar is a tool for arguing precisely — use it
  in PR review too.
- Disclose uncertainty. "I'm not sure" is more useful than
  performed confidence.
- Acknowledge when feedback changes your view.
- Keep PR discussions focused on the artefact at hand. If a
  broader concern surfaces, open a separate issue.

## Unacceptable behaviour

- Personal attacks, harassment, or discriminatory language.
- Bad-faith argumentation — moving the goalposts, refusing
  to engage with specific cited counterevidence, repeating
  rejected claims without addressing the counter.
- Posting credentials, private communications, or other
  people's personal data without consent.
- Sustained disruption of discussions or maintainer time.

## Enforcement

The maintainer reads every issue and PR comment. If a
contributor's behaviour falls outside the expected
range, the maintainer will:

1. First, address it in the thread directly.
2. If it continues, lock the thread and email the
   contributor.
3. As a last resort, ban from the repository.

There is no formal committee — this is a small project,
the enforcement is the maintainer's judgment, and the
appeal route is "open a separate issue or email the
maintainer."

## Scope

This code of conduct applies to interactions in this
repository (issues, PRs, discussions) and any project
spaces (e.g., a future Discord / Matrix / mailing list if
those ever exist).

## Attribution

This code of conduct is custom-written for this project,
informed by the Contributor Covenant tradition but
intentionally shorter. Most "CoC violations" are people
arguing badly; the rules above name what good arguing
looks like, and the rest follows.
